# 深度判断规则

## 总原则

```text
对话证据 60%
教材重要性 40%
```

## orientation

绪论、概览、过渡。仍需讲清是什么、为什么、例子和后续关系。

## standard

普通知识点。覆盖定义、直觉、机制、例子、易错点和边缘情况。

默认善用 Markdown 结构；涉及公式、复杂度、递推式或度量时使用 LaTeX，并解释符号含义。

## deep-concept

噪声、缺失值、数据质量、预处理、相似度、维归约、特征选择、过拟合等核心概念。尽可能详细。

应讲清背景、定义、直觉、分类、判断标准、方法比较、常见误区和实际用途。数据挖掘类主题按需补充带注释的数据集或外部参考。

## algorithm-or-math

算法、公式、度量、推导、评价指标。覆盖输入输出、变量、步骤、算例、适用条件、复杂度和错误。

必须使用 LaTeX 表达关键公式、递推式和复杂度。数据结构与算法主题按需补充带注释的练习题、LeetCode 链接、可视化工具或权威参考。

## 对话驱动升级

出现反复追问、明确困惑、多次纠正、额外例题需求、代码困难或边缘情况时升级一级。

## 教材兜底

教材核心知识即使未在对话中出现，也不得降为浅层摘要。
